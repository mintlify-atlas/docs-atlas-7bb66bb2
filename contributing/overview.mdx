Contributions to Accrescent are welcome. This page covers the channels and resources you need to get oriented before diving in.

<Note>
  Accrescent is currently in early alpha. The codebase, APIs, and processes are still evolving. Expect things to change.
</Note>

## Get help

The [#accrescent:matrix.org](https://matrix.to/#/#accrescent:matrix.org) Matrix room is the primary place for development discussion and questions. If you are unsure where to start, ask there first.

The lead developer, [@lberrymage:matrix.org](https://matrix.to/#/@lberrymage:matrix.org), is available on Matrix and welcomes direct messages as well.

<Tip>
  Questions asked on Matrix are visible to the whole community and receive faster responses than GitHub issues. Prefer Matrix for open-ended questions and design discussions.
</Tip>

## Find work

The [public roadmap](https://github.com/orgs/accrescent/projects/4/views/2) lists the project's current priorities and planned work. If you want to contribute but are not sure what to work on, the roadmap is the best starting point.

## Report bugs and suggest features

Each Accrescent repository has its own issue tracker. Use the tracker in the repository most relevant to your report.

If your issue spans multiple repositories, relates to infrastructure, or you are unsure where it belongs, open it in the [meta repository](https://github.com/accrescent/meta). Issues are transferred to the appropriate repository when necessary.

## Report vulnerabilities

Report all security vulnerabilities according to Accrescent's security policy, which is documented in `SECURITY.md` at the root of the main repository. Do not open public GitHub issues for security vulnerabilities.

## Translations

Accrescent uses [Hosted Weblate](https://hosted.weblate.org/engage/accrescent/) for translations. If you would like to help translate the app into your language, you can contribute directly through Weblate without any local build setup.
